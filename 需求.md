# iPaper 需求文档

## 项目概述

一个论文阅读工具，支持从 arXiv 下载论文，自动翻译成中文，并提供 LLM 辅助阅读功能。

---

## 功能1：爬取 arXiv 论文

| 需求项 | 状态 | 结论 |
|--------|------|------|
| 输入方式 | ✅ 已确定 | 同时支持完整 URL 和 arXiv ID（如 `2301.12345`） |
| LaTeX 缺失处理 | ✅ 已确定 | 只保留英文 PDF，界面提示"无法翻译" |
| 论文管理 | ✅ 已确定 | 需要论文库来管理已下载的论文 |
| 批量导入 | ✅ 已确定 | 暂不支持 |

| 需求项 | 状态 | 结论 |
|--------|------|------|
| 翻译引擎 | ✅ 已确定 | 使用 LLM Agent 进行翻译 |
| 翻译方式 | ✅ 已确定 | Interleave 逐段翻译：读一段→翻译→读下一段→翻译，保持完整上文可见 |
| 翻译粒度 | ✅ 已确定 | 以段落为单位 |
| 数学公式 | ✅ 已确定 | 保持原样不翻译 |
| 图表/表格 | ✅ 已确定 | 只翻译必要的 caption |
| 代码块/算法框 | ✅ 已确定 | 只翻译必要的注释 |
| 人名 | ✅ 已确定 | 不翻译 |
| 参考文献 | ✅ 已确定 | 不翻译（保持学术规范） |
| 源码可编译性 | ✅ 已确定 | 必须保持 LaTeX 源码可编译 |
| 翻译完整性检查 | ✅ 已确定 | 翻译完成后检查是否有漏翻译的章节和文件 |
| 编译工具 | ✅ 已确定 | 使用 xelatex 编译成 PDF |
| 并行翻译 | ✅ 已确定 | 内容较多时支持多 sub-agent 并行翻译 |
| 编译失败处理 | ✅ 已确定 | 混合方案：先尝试自动修复，失败则回退到英文原文 |
| 术语一致性 | ✅ 已确定 | 不特别处理，依赖 LLM 上下文理解自然保持一致 |
| 翻译进度显示 | ✅ 已确定 | 需要，前端显示翻译进度 |
| 论文摘要生成 | ✅ 已确定 | 翻译完成后，Agent 提取论文标题 + 生成简短介绍，用于论文库缩略显示 |

### 翻译流程详细说明

```
┌─────────────────────────────────────────────────────────────┐
│                    翻译 Agent 工作流程                        │
├─────────────────────────────────────────────────────────────┤
│  1. 读取 LaTeX 项目结构，识别所有需要翻译的 .tex 文件          │
│  2. 对于每个文件，采用 interleave 方式逐段翻译：               │
│     ┌────────────────────────────────────────┐              │
│     │  读取段落 1 → 翻译段落 1                 │              │
│     │  读取段落 2（上文可见）→ 翻译段落 2       │              │
│     │  读取段落 3（上文可见）→ 翻译段落 3       │              │
│     │  ...                                    │              │
│     └────────────────────────────────────────┘              │
│  3. 内容较多时，可启动多个 sub-agent 并行翻译不同章节/文件     │
│  4. 翻译完成后，检查所有章节和文件是否完整翻译                 │
│  5. 使用 xelatex 编译生成中文 PDF                            │
│  6. 提取论文标题（中英文）+ 生成简短摘要，写入 meta.json       │
└─────────────────────────────────────────────────────────────┘
```

### 翻译规则

| 内容类型 | 处理方式 |
|----------|----------|
| 正文文字 | 全部翻译成中文 |
| 数学公式 | 保持原样 |
| 图片/表格 caption | 翻译 |
| 代码块 | 保持原样，仅翻译注释 |
| 算法框 | 保持原样，仅翻译注释 |
| 人名 | 不翻译 |
| 参考文献 | 不翻译 |

### 数据存储结构

```
~/.ipaper/
├── config.json                 # 全局配置
├── papers/
│   ├── 2301.12345/            # 按 arXiv ID 组织
│   │   ├── meta.json          # 论文元信息（见下方详细说明）
│   │   ├── paper_en.pdf       # 英文 PDF
│   │   ├── paper_zh.pdf       # 中文 PDF
│   │   ├── source/            # LaTeX 源码
│   │   ├── chat_history.json  # 对话历史
│   │   └── position_map.json  # 中英文位置映射
│   └── 2302.67890/
│       └── ...
└── index.json                  # 论文索引（快速列表）
```

### meta.json 结构

```json
{
  "arxiv_id": "2301.12345",
  "title": "Attention Is All You Need",
  "title_zh": "注意力机制是你所需要的一切",
  "summary": "本文提出了 Transformer 架构，一种完全基于注意力机制的序列转换模型...",
  "authors": ["Ashish Vaswani", "Noam Shazeer", "..."],
  "download_time": "2024-01-15T10:30:00Z",
  "has_latex": true,
  "translation_status": "completed",  // "pending" | "translating" | "completed" | "failed"
  "translation_progress": 100         // 0-100，用于进度显示
}
```
