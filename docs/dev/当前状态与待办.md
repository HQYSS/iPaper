## 已知问题 (Bugs)

### P0 - 影响核心体验

| 问题 | 描述 | 根因分析 |
|------|------|---------|
| ~~搜索跳转后不居中~~ | ✅ 已修复。跳转后淡出 → 轮询等待滚动停止且高亮元素可见 → 居中 → 淡入 | 根因：1) `querySelector` 找到虚拟滚动保留的旧高亮元素 2) `scrollIntoView` 居中到窗口而非 PDF 容器。修复：`querySelectorAll` 取可见区域内最近元素 + 手动 `scrollTop` |

## 部署架构

```
用户浏览器（Mac Chrome / iPhone Safari）
        │
        ▼ HTTPS (port 443)
    ┌─────────┐
    │  Xray   │  ← TLS 终结，VPN 流量正常代理
    │         │     非 VPN 流量 fallback ↓
    └────┬────┘
         ▼ (port 8080, localhost)
    ┌─────────┐
    │  Nginx  │  ← /ipaper/ → 前端静态文件 (dist/)
    │         │     /ipaper/api/ → 反向代理到后端 (:3000)
    │         │     /disk/ → filebrowser
    │         │     /alist/ → alist
    │         │     iPaper 认证由应用层 JWT 处理
    └─────────┘
         ▼ (port 3000, localhost)
    ┌─────────┐
    │ FastAPI  │  ← systemd 管理 (ipaper-backend.service)
    │ Backend  │     venv 隔离 Python 依赖
    └─────────┘
```

### 部署操作速查

```bash
# 本地改代码后同步到服务器

cd .. && tar czf /tmp/dist.tar.gz -C frontend dist/

```

## 开发注意事项

### 前端修改须知

1. **@react-pdf-viewer 的虚拟滚动**：该库使用虚拟滚动渲染 PDF，很多常规的 CSS 布局技巧不适用。操作内部 DOM 时需通过 JS 查找实际的滚动容器
2. **插件实例化**：`searchPlugin()` 等工厂函数内部调用了 React hooks，它们本身就是 hooks，必须在组件顶层直接调用（不能包在 `useMemo`/`useState`/`useRef` 中）。每次渲染都会调用，但通过 React hooks 机制保持内部状态
3. **选中文字交互**：mouseup/mousedown 监听需限定在各自的容器 ref 内，否则 PDF 和 Chat 的引用按钮会互相干扰
4. **PDF Worker**：使用 CDN 地址 `https://cdnjs.cloudflare.com/ajax/libs/pdf.js/3.11.174/pdf.worker.min.js`，需确保版本与 pdfjs-dist 一致
5. **生产构建路径**：`vite.config.ts` 中 build 模式 `base: '/ipaper/'`，`api.ts` 中 API_BASE 基于 `import.meta.env.BASE_URL` 动态计算
6. **前端交互先自测**：能跑前端就先跑前端，能接浏览器 / Electron 就先复现和回归；不要只靠静态代码阅读判断交互问题已修复
7. **iPad Safari 兼容**：ReactMarkdown 在 iPad Safari（iOS 15.x）上会导致 WebKit 崩溃（根因不明），触屏设备的 Markdown 渲染必须走 `marked` + KaTeX MathML 路径。正则表达式禁止使用 lookbehind `(?<!...)` 语法（iOS 15.x 不支持，会直接崩溃）
8. **窄屏适配**：≤ 1024px 时桌面布局自动折叠侧边栏和对话面板；触屏设备上不自动 focus 输入框（防止弹键盘改变视口）
9. **三段布局路由**：`App.tsx` 在认证后用 `useDeviceLayout()` 选择 `MobileLayout`（短边 < 768px）或桌面三栏。`detectDeviceLayout` 用 `Math.min(window.innerWidth, window.innerHeight)` 判断，避免 iPhone 横屏（844×390）被当成平板。新增移动端 hooks 见 `frontend/src/hooks/`：`useDeviceLayout` / `useIsTouchDevice` / `useViewportOrientation` / `useVisualViewportHeight`，`useThemeMode` 抽离了主题副作用便于多布局复用
10. **PWA 资产**：图标在 `frontend/public/icons/`（生成脚本 `/tmp/ipaper-icon-design/generate_icons.py`，源图 `assets/m1-letter-i.png`）；改图标后必须重新跑脚本生成全套尺寸（含 maskable），并把 `frontend/public/sw.js` 的 `CACHE_NAME` 加一档版本号

### 后端修改须知

0. **论文下载已异步化**：`POST /api/papers` 只同步查 arXiv metadata 并返回占位 meta（`download_status=downloading`），实际 PDF 下载由 `asyncio.create_task` 在后台跑；同步 blocking 调用（`arxiv.Client().results()` / `paper.download_pdf()`）都用 `asyncio.to_thread` 丢线程池，不阻塞 FastAPI 事件循环。后台下载失败会按 2s/4s/8s 指数退避重试 3 次，仍失败则 `download_status=failed` + `download_error`。sync manifest 会过滤掉 `download_status != "ready"` 的论文，避免把"空架子"推上云端；`sync_service.delete_paper` 会先调 `arxiv_service.cancel_download` 取消正在跑的后台 task
1. **LLM 客户端懒加载**：`LLMService.client` 属性检测 API Key 变化后会重建客户端
2. **API 提供商**：OpenRouter（`https://openrouter.ai/api/v1`），通过 OpenAI Python SDK 调用
3. **PDF 上传方式**：PDF ≤ 15MB 时通过 `file` content type 发送 base64 编码的 PDF；PDF > 15MB 时自动用 PyMuPDF 逐页渲染为 JPEG（150 DPI, quality 85），以多张 `image_url` 发送
4. **Reasoning tokens**：通过 `extra_body={"reasoning": {"effort": "medium"}}` 启用 Gemini 思考模式，reasoning 在流式过程中累积并存入 `ChatMessage.reasoning` 字段，下一轮对话时传回模型。⚠️ Gemini 3.1 Pro 的 reasoning tokens 也会计入 `max_tokens`，默认值已提到 32768，调低前需评估长回答被截断的风险
5. **arXiv 速率限制**：频繁请求 arXiv API 会被限流（HTTP 429），建议添加延迟和重试
6. **SSE 流格式**：严格遵循 `data: {json}\n\n` 格式，前端按行解析
7. **画像分析使用独立模型**：`ProfileAnalysisConfig` 配置 `anthropic/claude-opus-4.6`（OpenRouter），与讲解模型分离
8. **LLM 错误日志**：`APIStatusError` 等异常会通过 `logger.error` 记录完整的上游错误信息（含 OpenRouter 返回的 message），前端也会显示具体原因而非泛化提示
9. **用户消息即时持久化**：`POST /chat/{id}/{sid}` 收到用户消息后立即保存到文件，不等 LLM 回复完成。即使 LLM 出错或用户停止生成，刷新后消息不丢失
10. **未发送草稿按会话持久化**：输入框文本和待发送引用通过 `/history/draft` 单独保存到会话 JSON 的 `draft` 字段；切换会话前会强制保存一次，发送成功后清空

### 服务器须知

1. **内存受限**：服务器仅 472MB，不能在服务器上执行 `npm run build`，必须本地构建
2. **Xray 不能动**：443 端口由 Xray 管理，Nginx 只监听 8080（localhost），改 Nginx 配置前必须 `nginx -t` 测试
3. **Python 虚拟环境**：后端依赖在 `/home/admin/iPaper/backend/venv/` 中，不污染系统 Python
