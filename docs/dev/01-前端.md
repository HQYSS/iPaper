## 页面布局

`App.tsx` 定义了三栏布局：

```
┌──────────┬────────────────────────┬──────────────┐
│          │                        │              │
│  论文库   │     PDF 阅读器          │  AI 对话面板  │
│  w-64    │     flex-1             │  w-96        │
│          │                        │              │
│ PaperLib │     PdfViewer          │  ChatPanel   │
│          │                        │              │
└──────────┴────────────────────────┴──────────────┘
```

- 左侧论文库固定 `256px`，中间 PDF 阅读器占据剩余空间
- 右侧讲解面板默认 `480px`，支持拖拽调整，最小 `320px`、最大不超过容器宽度的 `50%`
- 右侧讲解面板宽度会按比例记忆到 `localStorage`，窗口尺寸变化时按比例恢复
- 只有选中论文后才显示 PDF 阅读器和对话面板
- 论文库列表按添加时间（`download_time`）显示；该顺序在在线接口返回和离线缓存回退时保持一致
- 论文库和讲解面板均可折叠：`Cmd+B` 切换左侧论文库，`Cmd+L` 切换右侧讲解面板
- 支持全局论文快速切换：`Cmd+P` / `Ctrl+P` 打开 `PaperQuickSwitcher`，单论文模式下按最近打开顺序（MRU）列出“其他论文”，串讲模式下只列出当前串讲内的其他论文；最近顺序持久化到 `localStorage`
- 支持全局添加论文：`Cmd+N` / `Ctrl+N` 打开 `AddPaperModal`；侧边栏的 `+` 按钮、欢迎态的「添加论文」按钮也指向同一个弹窗。输入支持 arXiv ID、arXiv URL 或普通 PDF URL。`AddPaperModal`、`PaperQuickSwitcher` 任一打开时，全局快捷键（`Cmd+B/L/P/N`）暂时失效，避免穿透到背后的布局；当 `SettingsModal` 或画像面板打开时，仅 `Cmd+N` 被屏蔽，其他切换快捷键仍可用
- 选中论文后左侧论文库会自动收起；收起状态下鼠标移到最左侧触发区会临时展开，移出后自动收回
- 支持全局外观主题：`浅色 / 深色 / 跟随系统`，通过 `document.documentElement.dark` class 驱动

### 移动端三视图布局 (`MobileLayout`)

短边 < 768px（iPhone 各方向、小尺寸 PWA 启动窗）会走 `frontend/src/components/MobileLayout/`，整体形如：

```
┌────────────────────────┐
│ ☰  论文标题      +     │   顶栏（菜单 / 标题 / 添加）
├────────────────────────┤
│                        │
│   单视图：论文库 /      │
│         阅读 /         │   PaperLibrary / PdfViewer / ChatPanel
│         AI             │   全屏铺满，互不并存
│                        │
├────────────────────────┤
│  📚      📄      ✨    │   底部 Tab：论文库 / 阅读 / AI
└────────────────────────┘
```

- 顶栏菜单按钮打开 `MobileMenu`（左侧滑出 Sheet，集中放设置 / 阅读进化 / 退出登录）
- 选中论文后自动跳到「阅读」Tab；底栏切到「AI」时复用桌面 `ChatPanel`
- `PaperLibrary` 在移动端传 `hideBottomActions` 隐藏底部"串讲模式 / 设置"按钮，避免与底部 Tab 重叠（这两个动作在移动端分别由「论文库 Tab 长按」和「菜单 Sheet」承载）
- 容器高度用 `var(--visual-vh, 100dvh)`，配合 `useVisualViewportHeight` 把 `window.visualViewport.height` 写到 CSS 变量，键盘弹起时输入框始终贴在键盘上沿
- 安全区由 `env(safe-area-inset-*)` + `viewport-fit=cover` 兜住刘海/Home Indicator
- App 路由层（`App.tsx::AuthenticatedRouter`）用 `useDeviceLayout()` 决定渲染 `MobileLayout` 还是桌面三栏；`useThemeMode()` 把主题副作用抽出来给两套布局共用，避免逻辑双写

## 组件详解

### PaperLibrary (`components/PaperLibrary/index.tsx`)

论文库侧边栏。

**Props：**
```typescript
interface PaperLibraryProps {
  onOpenSettings?: () => void
  onOpenAddPaper?: () => void
}
```

**功能：**
- 显示已下载论文列表（标题 + 摘要缩略）
- 列表按添加时间（`download_time`）升序显示，避免 IndexedDB 读取时退化成按 `arxiv_id` 键顺序展示
- 顶部 `+` 按钮直接打开 `AddPaperModal`（不再在侧边栏内联展开输入框），交互和 `Cmd+N` 快捷键、欢迎态按钮完全一致；添加 arXiv 论文时按钮 loading 只等后端查 arXiv metadata（~1-2s），添加普通 PDF URL 时只等后端写入占位 meta，返回后立即结束，英文 PDF 在后台下载
- 按 `download_status` 渲染卡片状态：
  - `downloading`：标题下方显示 `<Loader2>` + "正在下载英文 PDF…"
  - `failed`：标题灰化 + 红色 `<AlertTriangle>` + 错误信息；卡片右上角额外露出 `<RotateCw>` 重新下载按钮，arXiv 论文用 `arxiv_id` 重试，普通 PDF URL 用 `source_url` 重试
  - `ready`：正常
- 串讲模式下 `downloading/failed` 的论文不允许加入串讲（点击 checkbox 会弹 toast 提示）
- 删除论文；`sync_service.delete_paper` 会同步 cancel 后台下载 task
- 点击选中论文
- 底部齿轮按钮打开设置弹窗

**依赖的 Store：** `usePaperStore`

**对外交互：** 通过 `paperStore.selectPaper()` 设置当前选中论文，驱动 PdfViewer 和 ChatPanel 加载

**下载状态轮询：** `paperStore.ensureDownloadPolling()` 在有 `download_status === 'downloading'` 的论文时每 2s 拉一次 `GET /papers`，全部 ready/failed 后停。每次轮询会同步更新 `selectedPaper` 引用，让 App.tsx 感知状态变化自动切换阅读面板 / 占位面板。

**下载占位与失败面板（在 `App.tsx` 中间主区域）：** 若当前 `selectedPaper.download_status` 为
- `downloading` → 显示"正在下载英文 PDF…"占位（中间 loader + 提示文案），同时右侧 ChatPanel 被隐藏（因为 LLM 依赖 PDF）
- `failed` → 显示红色警告面板 + "重新下载"按钮，arXiv 论文用 `arxiv_id` 重试，普通 PDF URL 用 `source_url` 重试
- `ready` → 正常渲染 PdfViewer 与 ChatPanel

### PdfViewer (`components/PdfViewer/index.tsx`)

PDF 阅读器，是前端最复杂的组件。

**Props：**
```typescript
interface PdfViewerProps {
  paperId: string         // 论文 ID，用于构造 PDF URL；普通 PDF URL 论文为 pdf_<hash>
  sourceType?: 'arxiv' | 'pdf_url'
  mobileMode?: boolean    // 移动端模式：见下方"移动端简化模式"
}
```

**功能模块：**

#### 1. PDF 渲染
- 使用 `@react-pdf-viewer/core` 的 `<Viewer>` 组件
- Worker URL：`https://cdnjs.cloudflare.com/ajax/libs/pdf.js/3.11.174/pdf.worker.min.js`
- PDF URL：通过 `getPdfUrl(paperId)` → `/api/papers/{id}/pdf`

#### 2. 插件系统
使用了 4 个 `@react-pdf-viewer` 插件：

| 插件 | 用途 | 提供的组件/方法 |
|------|------|----------------|
| `searchPlugin` | 文内搜索 | `highlight()`, `jumpToNextMatch()`, `jumpToPreviousMatch()`, `clearHighlights()` |
| `bookmarkPlugin` | 目录导航 | `<Bookmarks />` 组件 |
| `pageNavigationPlugin` | 页码 | `<CurrentPageLabel />` 组件 |
| `zoomPlugin` | 缩放 | `<ZoomIn />`, `<ZoomOut />`, `<CurrentScale />` 组件 |

#### 3. 搜索功能 (SearchBox 子组件)
- `Cmd+F` 呼出搜索框（浮动在右上角）
- `Enter` 跳转下一个匹配，`Shift+Enter` 上一个，`Esc` 关闭
- 显示当前匹配序号 / 总匹配数
- 跳转后将匹配项滚动到视口中央：
  - 淡出 PDF 内容 → 每 80ms 轮询检测滚动是否停止且高亮元素是否出现在可视区域 → 手动调整 `.rpv-core__inner-pages` 的 `scrollTop` 居中 → 淡入
  - 使用 `querySelectorAll('.rpv-search__highlight--current')` 获取所有高亮元素，取可视区域内最近的（虚拟滚动会保留旧页面的高亮 DOM）
  - `cleanupRef` 确保连续按 Enter 时取消上一次未完成的轮询

#### 4. 缩放后居中
- 使用 `handleZoomWithFade()` 包装缩放操作
- 流程：淡出(0.1s) → 执行缩放 → 滚动居中 → 淡入(0.1s)
- 通过淡入淡出避免「先渲染再滚动」导致的视觉抖动
- 用户通过底部 `+ / -` 按钮调整的缩放比例会记忆到 `localStorage`，刷新页面和切换论文后自动恢复
- 切换 `EN / 中文 / 双语` PDF 时，会临时保存当前缩放和纵向滚动比例，在新 PDF 加载完成后恢复

#### 5. 选中文字引用
- 使用全局 `selectionchange` 监听 PDF text layer 的真实选区变化，而不是只依赖 `mouseup`
- 选中后在文字上方显示「引用到对话」浮动按钮
- 点击后调用 `chatStore.addQuote(text, 'pdf')`，并自动把焦点切回 `ChatPanel` 输入框
- 按钮通过 `data-quote-button` 和 `onMouseDown(e.preventDefault())` 避免点击时先清掉选区
- 选区检测会校验当前 `Selection` 是否仍位于 `PdfViewer` 的 `.rpv-core__inner-pages` 内，避免和对话区引用互相干扰

#### 6. 底部浮动工具栏
- 圆角半透明工具栏，包含：目录按钮、搜索按钮、引用返回、缩小/放大/百分比、页码跳转、实验性调光、语言切换
- 工具栏和书签侧栏已支持 `dark:` 外壳样式；PDF 页面本体保持原始白底
- 当 `sourceType === 'pdf_url'` 时隐藏 `EN / 中文` 切换和翻译进度，强制使用英文 PDF；普通 PDF URL 不走 hjfy.top 翻译

#### 7. PDF 内部跳转返回
- `PdfViewer` 会在目录点击、文内链接、搜索跳转和页码跳转前记录当前位置
- 对目录和文内链接这类异步导航，先记录候选位置，再等 `.rpv-core__inner-pages` 真的发生滚动后压入返回栈，避免“点击了但没跳转”时写入脏数据
- 返回栈按 `paperId + pdfLang` 缓存在组件外部，切换 `EN / 中文` 或串讲中的 PDF tab 时不会把当前语言对应的返回历史串到别的视图
- 底部“返回引用位置”按钮始终回到最近一次有效跳转前的位置

#### 8. 实验性调光
- 底部工具栏提供临时 `调光` 面板，用于对比两种 PDF 压暗方案
- `Overlay`：在 PDF 可视区域上覆盖一层半透明深色遮罩，支持多档遮罩强度
- `Brightness`：对 PDF 可视区域施加 `filter: brightness(...)`，支持多档亮度
- 两种调光方案都只作用于 PDF 可视区域，不改变底部工具栏本身的颜色
- 调光模式和档位都会记忆到 `localStorage`

#### 9. 移动端简化模式 (`mobileMode`)
`MobileLayout` 渲染 `PdfViewer` 时传 `mobileMode`，触发以下差异：
- **隐藏全部桌面交互**：底部浮动工具栏（缩放 +/-/百分比、页码跳转、调光、目录、引用返回）和左侧书签侧栏全部不渲染
- **顶部极简工具栏**：在 PDF 区域上方插入一行 `h-10` 工具栏，左：搜索按钮，右：翻译进度（如果有）+ `EN / 中文` 切换胶囊
- **搜索框样式切换**：`SearchBox` 接收 `mobile` prop，从顶右角浮窗改为顶部全宽下拉（`top-2 left-2 right-2`），输入框 `flex-1`
- **fit-width 默认**：`onDocumentLoad` 直接调 `zoomPluginInstance.zoomTo(SpecialZoomLevel.PageWidth)`，忽略 `prefs.getPdfScale()`（桌面端记忆值），避免移动端被强制按桌面 170% 显示导致横向溢出
- **不恢复滚动位置**：移动端每次进入论文从顶部开始，符合手机阅读习惯
- **缩放交还系统**：`frontend/index.html` 的 viewport meta 已去掉 `user-scalable=no, maximum-scale=1`，靠 iOS 自带 pinch-to-zoom；为防止 iOS focus input 时自动放大整页，`index.css` 给 input/textarea 兜底 `font-size: 16px`

### ChatPanel (`components/ChatPanel/index.tsx`)

AI 对话面板。

**Props：**
```typescript
interface ChatPanelProps {
  paperId?: string                              // 单论文模式
  crossPaperSessionId?: string                  // 串讲模式（二选一）
  onCollapse?: () => void
  onPaperLinkClick?: (paperId: string) => void  // 串讲中点击 [[id]] 链接
  onExitCrossChat?: () => void                  // 退出串讲
}
```

**功能模块：**

#### 1. 消息显示
- 用户消息：右对齐，蓝色背景
- AI 消息：左对齐，灰色背景，使用 ReactMarkdown 渲染
- 支持 LaTeX 公式（remark-math + rehype-katex）
- 流式响应时逐字显示，智能自动滚动（用户在底部时跟随，上翻阅读时不强制拉回）
- 发送时会先在前端插入“用户消息 + assistant 占位消息”，并立刻清空该会话的持久化草稿，避免请求尚未真正发出时旧草稿在稍后重新出现
- 流式响应真正建立后，Store 会把当前会话快照写回历史；后续 chunk 以节流方式持续覆盖这条 assistant 消息
- 切换到其他论文、切换会话、切进/切出串讲时，会先中止当前流式请求，避免旧请求继续悬挂并污染新上下文；如果中止时还没有生成正文，会移除空 assistant 占位，只保留已发送的用户消息
- 加载会话历史时，如果尾部存在“空 assistant 占位”脏数据，会自动清理并回写，避免讲解面板永久卡成 `...`
- 对话滚动位置按会话隔离记忆：单论文模式使用 `paperId + sessionId`，串讲模式使用 `crossPaperSessionId`；切换论文、切换会话、切回历史会话时分别恢复各自位置，不串用滚动状态

#### 2. 引用系统（多段引用）

**引用来源：** PDF 选中文字 (`source: 'pdf'`) 或 对话选中文字 (`source: 'chat'`)

**支持多段引用：** Store 中 `quotes` 为数组，每次选中文字调用 `addQuote()` 追加，输入框上方渲染多个引用卡片（彩色左边框+图标区分来源，点击展开），每个带独立删除按钮

**引用后聚焦：** 每次通过 PDF 或对话区追加引用后，`ChatPanel` 会自动聚焦到底部输入框，并将光标移动到当前输入内容末尾，便于立即继续提问

**文字清理：** 选中文字经白名单过滤（只保留 ASCII、Latin、Greek、CJK、常用标点和数学符号），清除 PDF 特殊字体和 KaTeX 渲染产生的不可见/乱码 Unicode 字符

**发送方式：** 引用通过 API 的 `quotes` 字段结构化传递（`[{text, source}]`），消息内容只包含用户问题，不嵌入标记。发送时会将 `quotes` 挂到用户消息对象上，因此聊天区可立即显示引用块，刷新后从历史恢复时也能继续显示

**未发送草稿持久化：** 输入框文本和待发送引用不再只存在组件本地 state，而是作为当前会话的 `draft`（`{ input, quotes }`）保存在后端。输入过程中会 debounce 同步；切换单论文会话、切换串讲会话、刷新页面后重新进入时都会恢复；真正发送消息后会清空这份草稿

**超大 PDF 页码选择：** 当后端判断“PDF 转成图像后仍超过 20MB”时，`ChatPanel` 会弹出 `PageSelectionModal`。弹窗按论文分组展示，每篇论文都支持通过输入框录入 `12` 或 `12-18` 这类单页/范围，并在按 `Enter` 后逐条加入列表；单论文和串讲共用同一套交互。如果第一次选择的范围仍然过大，后端会把已选范围带回，用户继续缩小即可

**页码选择会话记忆：** 当前会话已经确认过的 `page_selections` 会跟随草稿一起保存；后续同一会话继续提问时默认复用，不会每轮都重新要求选页

**渲染优先级：** 用户消息渲染时优先读取 `message.quotes` 显示引用块；仅对旧历史消息保留 `parseMessageWithQuote()` 的 markup 兼容分支

**旧消息兼容：** `parseMessageWithQuote()` 保留用于渲染历史消息中的 `<<<QUOTE>>>` 标记

#### 3. 对话中选中文字引用
- 对话区域也有独立的 `mouseup`/`mousedown` 监听
- 选中文字后显示「引用到对话」按钮（使用 `data-chat-quote-button` 区分）
- 与 PDF 引用按钮互不干扰（各自限定在自己的容器 ref 内）
- 点击引用按钮后同样会自动把焦点切回底部输入框

#### 4. 错误提示
- Store 中 `error` 状态非空时，在消息列表底部渲染红色提示条（`AlertCircle` 图标 + 错误消息 + 关闭按钮）
- SSE `type: 'error'` 或 fetch 异常均会触发，同时移除空的 assistant 占位消息
- 后端按 OpenAI SDK 异常类型返回中文友好消息（Key 无效/额度不足/连接失败/服务端错误）
- SSE `type: 'truncated'` 不算错误：不清空消息、不 set `error`，只给 assistant 消息打 `truncated:true`，在气泡下方渲染黄色截断提示条，提醒用户可追问"请继续"

#### 5. 清空对话
标题栏右侧垃圾桶图标，confirm 确认后调用 `chatStore.clearHistory()`

#### 6. 页码选择弹窗
- 组件：`components/PageSelectionModal/index.tsx`
- 触发条件：`api.sendMessage()` / `api.sendCrossPaperMessage()` 收到后端 `409 + page_selection_required`
- 每张论文卡片独立录入范围，支持 Enter 添加、标签删除、一次性确认发送
- 串讲模式会把所有需要缩减的论文一次性列出，而不是一篇一篇打断

#### 7. 串讲启动与加论文行为
- `initCrossPaperSession()` 只负责把前端切到串讲会话，不再自动发送首条“串讲分析”消息
- 串讲空态会明确提示“请先提问”，由用户决定先问什么、按什么角度对比
- 串讲中通过 `+` 添加论文时，默认只更新会话里的 `paper_ids`
- 只有用户在“想让 AI 接下来分析什么？”输入框里填写了问题，`addPaperToCrossChat()` 才会立刻发送该问题

### SettingsModal (`components/SettingsModal/index.tsx`)

## Hooks (`src/hooks/`)

| Hook | 用途 |
|------|------|
| `useDeviceLayout()` | 返回 `'mobile' \| 'tablet' \| 'desktop'`，监听 `resize` / `orientationchange`，按短边 (`Math.min(w, h)`) 判定，避免 iPhone 横屏被当成平板。`App.tsx` 用它选布局 |
| `useIsTouchDevice()` | 检测是否触屏设备（`(pointer: coarse)` + `ontouchstart`），用于关闭 PDF / Markdown 上 hover-only 的高亮态 |
| `useViewportOrientation()` | 返回当前是否横屏，用于 iPad 上后续切分栏方向 |
| `useVisualViewportHeight()` | 监听 `window.visualViewport.resize`，把可视高度写到 CSS 变量 `--visual-vh`。**注意**：移动端容器主轴用 `height: 100svh` 而不是 dvh — Safari 工具栏动态收/展时 dvh 数值有滞后，会导致底部 Tab 一会儿被遮一会儿露空；svh 永远等于工具栏完全展开时的最小可视高度，最稳。`--visual-vh` 留给 ChatPanel 等需要键盘弹起精确感知的子组件按需消费 |
| `useThemeMode()` | 抽离主题（浅 / 深 / 系统）的状态 + 副作用，让桌面布局与移动布局共用同一份逻辑 |

底层判定逻辑在 `services/env.ts`：`MOBILE_BREAKPOINT = 768` / `TABLET_BREAKPOINT = 1024`，`detectIsStandalonePwa()` 检测 `display-mode: standalone` 用于 PWA 模式调整。

### chatStore (`stores/chatStore.ts`)

使用 `getChatHistoryOffline()` / `listSessionsOffline()` / `listCrossPaperSessionsOffline()` 等离线增强函数加载数据，编辑/分支保存使用 `updateChatHistoryOffline()` / `updateCrossPaperChatHistoryOffline()`。

```typescript
interface ChatStore {
  messages: ChatMessage[]        // 当前论文的对话消息
  isLoading: boolean
  isStreaming: boolean           // 是否正在流式接收
  error: string | null
  currentPaperId: string | null  // 当前关联的论文 ID
  draftInput: string             // 当前会话未发送输入
  quotes: QuoteItem[]            // 待发送的引用片段数组
  pendingPageSelection: PendingPageSelectionRequest | null
  pageSelectionsByConversation: Record<string, PaperPageSelectionInput[]>

  loadHistory(paperId, sessionId): Promise<void>         // 加载对话历史（离线增强）
  sendMessage(paperId, sessionId, message, quotes?, pageSelections?): Promise<void>  // 发送消息（SSE 流式，需在线）
  clearHistory(paperId, sessionId): Promise<void>        // 清空历史
  clearError(): void
  setDraftInput(input): void             // 更新未发送输入
  addQuote(text, source): void           // 追加一条引用
  removeQuote(index): void               // 删除指定引用
  clearQuotes(): void                    // 清空所有引用
  saveDraft(paperId, sessionId, input, quotes): Promise<void>  // 持久化当前草稿
  submitPageSelections(pageSelections): Promise<void>    // 确认页码后重发消息
  dismissPageSelection(): void                           // 关闭页码选择弹窗

  // 串讲
  isCrossPaperMode: boolean
  crossPaperIds: string[]
  initCrossPaperSession(session): Promise<void>  // 初始化串讲会话，不自动触发首问
  sendCrossPaperMessage(sid, msg, quotes?, pageSelections?): Promise<void>
  saveCrossPaperDraft(sid, input, quotes): Promise<void>
  addPaperToCrossChat(sid, ids, note): Promise<void>
  exitCrossPaperChat(): void
}
```

消息发送流程：
1. 添加用户消息到 `messages`
2. 添加空的 assistant 消息（占位）
3. 通过 `api.sendMessage()` 建立 SSE 连接；连接建立后先将当前会话历史写回后端和 IndexedDB，保证占位 assistant 可恢复
4. 接收 `type: 'chunk'` 事件，追加到 assistant 消息的 content，并按节流频率持续同步最新历史快照
5. 接收 `type: 'truncated'` 事件（因 `max_tokens` 被截断时出现）：把最后一条 assistant 消息的 `truncated` 标为 `true`，`ChatPanel` 会在该消息下方渲染一条黄色提示条
6. 接收 `type: 'done'` 事件，结束流式并做最后一次历史落盘
7. 接收 `type: 'error'` / `AbortError` 事件时，移除尾部空 assistant 占位，并将保留下来的历史重新写回
7. `loadSessions()` 对“首次自动创建默认会话”做了按 `paperId` 的单飞去重，避免 React `StrictMode` 或切换时序导致同一篇论文重复生成多个 `自动讲解` 会话壳；如果历史中已存在无消息的 `自动讲解` 空壳，会在加载会话列表时 best-effort 清理

## API 服务层 (`services/api.ts`) + 离线代理层

所有后端 API 调用封装在 `api.ts` 中。使用原生 `fetch`，无额外 HTTP 库。

离线增强层由 `cache.ts`（IndexedDB 封装）和 `offlineApi.ts`（离线代理）提供：
- `cache.ts`：使用 `idb` 库封装 IndexedDB，6 个 store（papers、pdfs、chats、sessions、preferences、pendingOps），含 LRU 淘汰（500MB）
- `offlineApi.ts`：为每个读取类 API 提供 `xxxOffline()` 函数，采用 network-first + cache-fallback 策略；写操作断网时排入队列，联网后自动重放；聊天历史写入成功后也会立即回写 IndexedDB，避免刷新后读到旧会话快照
- `api.ts` 末尾 re-export 所有 offlineApi 函数，消费侧可直接 `import { fetchPapersOffline } from './api'`

**基础路径：** `/api`（通过 Vite proxy 代理到后端 `localhost:3000`）

| 函数 | HTTP | 路径 | 说明 |
|------|------|------|------|
| `fetchPapers()` | GET | `/api/papers` | 获取论文列表 |
| `addPaper(input)` | POST | `/api/papers` | 添加论文 |
| `getPaper(id)` | GET | `/api/papers/{id}` | 获取论文详情 |
| `deletePaper(id)` | DELETE | `/api/papers/{id}` | 删除论文 |
| `getPdfUrl(id)` | - | `/api/papers/{id}/pdf` | 返回 PDF URL 字符串 |
| `getChatHistory(id, sid)` | GET | `/api/chat/{id}/{sid}/history` | 获取单论文会话历史 |
| `updateChatDraft(id, sid, draft)` | PUT | `/api/chat/{id}/{sid}/history/draft` | 更新单论文会话未发送草稿 |
| `clearChatHistory(id, sid)` | DELETE | `/api/chat/{id}/{sid}/history` | 清空单论文会话历史 |
| `sendMessage(id, sid, msg, quotes?, pageSelections?)` | POST | `/api/chat/{id}/{sid}` | 发送消息（SSE）；超大 PDF 会先抛出 `PageSelectionRequiredError` |
| `listCrossPaperSessions()` | GET | `/api/chat/cross-paper/sessions` | 串讲会话列表 |
| `createCrossPaperSession(ids)` | POST | `/api/chat/cross-paper/sessions` | 创建串讲会话 |
| `addPapersToCrossPaperSession(sid, ids)` | PUT | `/api/chat/cross-paper/sessions/{sid}/papers` | 添加论文到串讲 |
| `sendCrossPaperMessage(sid, msg, quotes?, pageSelections?)` | POST | `/api/chat/cross-paper/{sid}` | 串讲对话（SSE）；超大 PDF 会先抛出 `PageSelectionRequiredError` |
| `updateCrossPaperChatDraft(sid, draft)` | PUT | `/api/chat/cross-paper/{sid}/history/draft` | 更新串讲会话未发送草稿 |
| `getConfig()` | GET | `/api/config` | 获取配置 |
| `updateLLMConfig(config)` | PUT | `/api/config/llm` | 更新 LLM 配置 |

`sendMessage()` 和 `sendCrossPaperMessage()` 返回 `AsyncGenerator`，逐行解析 SSE 的 `data:` 行。若服务端返回 `409` 且 `detail.code === 'page_selection_required'`，`api.ts` 会改为抛出 `PageSelectionRequiredError`，由 `chatStore` 驱动页码选择弹窗。
