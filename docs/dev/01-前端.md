## 页面布局

`App.tsx` 定义了三栏布局：

```
┌──────────┬────────────────────────┬──────────────┐
│          │                        │              │
│  论文库   │     PDF 阅读器          │  AI 对话面板  │
│  w-64    │     flex-1             │  w-96        │
│          │                        │              │
│ PaperLib │     PdfViewer          │  ChatPanel   │
│          │                        │              │
└──────────┴────────────────────────┴──────────────┘
```

- 左侧论文库固定 `256px`，中间 PDF 阅读器占据剩余空间
- 右侧讲解面板默认 `480px`，支持拖拽调整，最小 `320px`、最大不超过容器宽度的 `50%`
- 右侧讲解面板宽度会按比例记忆到 `localStorage`，窗口尺寸变化时按比例恢复
- 只有选中论文后才显示 PDF 阅读器和对话面板
- 论文库和讲解面板均可折叠：`Cmd+B` 切换左侧论文库，`Cmd+L` 切换右侧讲解面板
- 选中论文后左侧论文库会自动收起；收起状态下鼠标移到最左侧触发区会临时展开，移出后自动收回
- 支持全局外观主题：`浅色 / 深色 / 跟随系统`，通过 `document.documentElement.dark` class 驱动

### PdfViewer (`components/PdfViewer/index.tsx`)

PDF 阅读器，是前端最复杂的组件。

**Props：**
```typescript
interface PdfViewerProps {
  paperId: string  // arXiv ID，用于构造 PDF URL
}
```

**功能模块：**

#### 1. PDF 渲染
- 使用 `@react-pdf-viewer/core` 的 `<Viewer>` 组件
- Worker URL：`https://cdnjs.cloudflare.com/ajax/libs/pdf.js/3.11.174/pdf.worker.min.js`
- PDF URL：通过 `getPdfUrl(paperId)` → `/api/papers/{id}/pdf`

#### 2. 插件系统
使用了 4 个 `@react-pdf-viewer` 插件：

| 插件 | 用途 | 提供的组件/方法 |
|------|------|----------------|
| `searchPlugin` | 文内搜索 | `highlight()`, `jumpToNextMatch()`, `jumpToPreviousMatch()`, `clearHighlights()` |
| `bookmarkPlugin` | 目录导航 | `<Bookmarks />` 组件 |
| `pageNavigationPlugin` | 页码 | `<CurrentPageLabel />` 组件 |
| `zoomPlugin` | 缩放 | `<ZoomIn />`, `<ZoomOut />`, `<CurrentScale />` 组件 |

#### 3. 搜索功能 (SearchBox 子组件)
- `Cmd+F` 呼出搜索框（浮动在右上角）
- `Enter` 跳转下一个匹配，`Shift+Enter` 上一个，`Esc` 关闭
- 显示当前匹配序号 / 总匹配数
- 跳转后将匹配项滚动到视口中央：
  - 淡出 PDF 内容 → 每 80ms 轮询检测滚动是否停止且高亮元素是否出现在可视区域 → 手动调整 `.rpv-core__inner-pages` 的 `scrollTop` 居中 → 淡入
  - 使用 `querySelectorAll('.rpv-search__highlight--current')` 获取所有高亮元素，取可视区域内最近的（虚拟滚动会保留旧页面的高亮 DOM）
  - `cleanupRef` 确保连续按 Enter 时取消上一次未完成的轮询

#### 4. 缩放后居中
- 使用 `handleZoomWithFade()` 包装缩放操作
- 流程：淡出(0.1s) → 执行缩放 → 滚动居中 → 淡入(0.1s)
- 通过淡入淡出避免「先渲染再滚动」导致的视觉抖动
- 用户通过底部 `+ / -` 按钮调整的缩放比例会记忆到 `localStorage`，刷新页面和切换论文后自动恢复
- 切换 `EN / 中文 / 双语` PDF 时，会临时保存当前缩放和纵向滚动比例，在新 PDF 加载完成后恢复

#### 5. 选中文字引用
- `mouseup` 事件监听选中文字
- 选中后在文字上方显示「引用到对话」浮动按钮
- 点击后调用 `chatStore.setQuotedText(text, 'pdf')`
- 使用 `data-quote-button` 属性防止点击按钮时清除选中

#### 6. 底部浮动工具栏
- 圆角半透明工具栏，包含：目录按钮、搜索按钮、引用返回、缩小/放大/百分比、页码显示、实验性调光、语言切换
- 工具栏和书签侧栏已支持 `dark:` 外壳样式；PDF 页面本体保持原始白底

#### 7. 实验性调光
- 底部工具栏提供临时 `调光` 面板，用于对比两种 PDF 压暗方案
- `Overlay`：在 PDF 可视区域上覆盖一层半透明深色遮罩，支持多档遮罩强度
- `Brightness`：对 PDF 可视区域施加 `filter: brightness(...)`，支持多档亮度
- 两种调光方案都只作用于 PDF 可视区域，不改变底部工具栏本身的颜色
- 调光模式和档位都会记忆到 `localStorage`

### ChatPanel (`components/ChatPanel/index.tsx`)

AI 对话面板。

**Props：**
```typescript
interface ChatPanelProps {
  paperId?: string                              // 单论文模式
  crossPaperSessionId?: string                  // 串讲模式（二选一）
  onCollapse?: () => void
  onPaperLinkClick?: (paperId: string) => void  // 串讲中点击 [[id]] 链接
  onExitCrossChat?: () => void                  // 退出串讲
}
```

**功能模块：**

#### 1. 消息显示
- 用户消息：右对齐，蓝色背景
- AI 消息：左对齐，灰色背景，使用 ReactMarkdown 渲染
- 支持 LaTeX 公式（remark-math + rehype-katex）
- 流式响应时逐字显示，智能自动滚动（用户在底部时跟随，上翻阅读时不强制拉回）
- 对话滚动位置按会话隔离记忆：单论文模式使用 `paperId + sessionId`，串讲模式使用 `crossPaperSessionId`；切换论文、切换会话、切回历史会话时分别恢复各自位置，不串用滚动状态

#### 2. 引用系统（多段引用）

**引用来源：** PDF 选中文字 (`source: 'pdf'`) 或 对话选中文字 (`source: 'chat'`)

**支持多段引用：** Store 中 `quotes` 为数组，每次选中文字调用 `addQuote()` 追加，输入框上方渲染多个引用卡片（彩色左边框+图标区分来源，点击展开），每个带独立删除按钮

**引用后聚焦：** 每次通过 PDF 或对话区追加引用后，`ChatPanel` 会自动聚焦到底部输入框，并将光标移动到当前输入内容末尾，便于立即继续提问

**文字清理：** 选中文字经白名单过滤（只保留 ASCII、Latin、Greek、CJK、常用标点和数学符号），清除 PDF 特殊字体和 KaTeX 渲染产生的不可见/乱码 Unicode 字符

**发送方式：** 引用通过 API 的 `quotes` 字段结构化传递（`[{text, source}]`），消息内容只包含用户问题，不嵌入标记。发送时会将 `quotes` 挂到用户消息对象上，因此聊天区可立即显示引用块，刷新后从历史恢复时也能继续显示

**未发送草稿持久化：** 输入框文本和待发送引用不再只存在组件本地 state，而是作为当前会话的 `draft`（`{ input, quotes }`）保存在后端。输入过程中会 debounce 同步；切换单论文会话、切换串讲会话、刷新页面后重新进入时都会恢复；真正发送消息后会清空这份草稿

**渲染优先级：** 用户消息渲染时优先读取 `message.quotes` 显示引用块；仅对旧历史消息保留 `parseMessageWithQuote()` 的 markup 兼容分支

**旧消息兼容：** `parseMessageWithQuote()` 保留用于渲染历史消息中的 `<<<QUOTE>>>` 标记

#### 3. 对话中选中文字引用
- 对话区域也有独立的 `mouseup`/`mousedown` 监听
- 选中文字后显示「引用到对话」按钮（使用 `data-chat-quote-button` 区分）
- 与 PDF 引用按钮互不干扰（各自限定在自己的容器 ref 内）
- 点击引用按钮后同样会自动把焦点切回底部输入框

#### 4. 错误提示
- Store 中 `error` 状态非空时，在消息列表底部渲染红色提示条（`AlertCircle` 图标 + 错误消息 + 关闭按钮）
- SSE `type: 'error'` 或 fetch 异常均会触发，同时移除空的 assistant 占位消息
- 后端按 OpenAI SDK 异常类型返回中文友好消息（Key 无效/额度不足/连接失败/服务端错误）

#### 5. 清空对话
标题栏右侧垃圾桶图标，confirm 确认后调用 `chatStore.clearHistory()`

## 状态管理 (Zustand Stores)

### paperStore (`stores/paperStore.ts`)

```typescript
interface PaperStore {
  papers: PaperListItem[]       // 论文列表
  selectedPaper: PaperListItem | null  // 当前选中论文
  isLoading: boolean
  error: string | null

  crossPaper: CrossPaperState           // 串讲状态

  fetchPapers(): Promise<void>          // 拉取论文列表
  addPaper(arxivInput: string): Promise<void>  // 添加论文（自动选中）
  deletePaper(paperId: string): Promise<void>  // 删除论文
  selectPaper(paper | null): void       // 选中论文（同时退出串讲状态）
  clearError(): void

  // 串讲
  enterCrossPaperMode(): void           // 进入多选模式
  exitCrossPaperMode(): void            // 退出串讲
  toggleCrossPaperSelection(id): void   // 切换选中（上限 5 篇）
  startCrossChat(): Promise<void>       // 创建串讲会话
  enterCrossChatSession(session): void  // 进入历史串讲
  setCrossPaperPdfTab(id): void         // 切换 PDF tab
  updateCrossPaperSessionPaperIds(ids): void  // 添加论文后同步
}
```

### chatStore (`stores/chatStore.ts`)

```typescript
interface ChatStore {
  messages: ChatMessage[]        // 当前论文的对话消息
  isLoading: boolean
  isStreaming: boolean           // 是否正在流式接收
  error: string | null
  currentPaperId: string | null  // 当前关联的论文 ID
  draftInput: string             // 当前会话未发送输入
  quotes: QuoteItem[]            // 待发送的引用片段数组

  loadHistory(paperId, sessionId): Promise<void>         // 加载对话历史
  sendMessage(paperId, sessionId, message, quotes?): Promise<void>  // 发送消息（SSE 流式）
  clearHistory(paperId, sessionId): Promise<void>        // 清空历史
  clearError(): void
  setDraftInput(input): void             // 更新未发送输入
  addQuote(text, source): void           // 追加一条引用
  removeQuote(index): void               // 删除指定引用
  clearQuotes(): void                    // 清空所有引用
  saveDraft(paperId, sessionId, input, quotes): Promise<void>  // 持久化当前草稿

  // 串讲
  isCrossPaperMode: boolean
  crossPaperIds: string[]
  initCrossPaperSession(session): Promise<void>  // 初始化 + 自动触发
  sendCrossPaperMessage(sid, msg, quotes?): Promise<void>
  saveCrossPaperDraft(sid, input, quotes): Promise<void>
  addPaperToCrossChat(sid, ids, note): Promise<void>
  exitCrossPaperChat(): void
}
```

消息发送流程：
1. 添加用户消息到 `messages`
2. 添加空的 assistant 消息（占位）
3. 通过 `api.sendMessage()` 建立 SSE 连接
4. 接收 `type: 'chunk'` 事件，追加到 assistant 消息的 content
5. 接收 `type: 'done'` 事件，结束流式
6. 接收 `type: 'error'` 事件，显示错误

## API 服务层 (`services/api.ts`)

所有后端 API 调用封装在此文件中。使用原生 `fetch`，无额外 HTTP 库。

**基础路径：** `/api`（通过 Vite proxy 代理到后端 `localhost:3000`）

| 函数 | HTTP | 路径 | 说明 |
|------|------|------|------|
| `fetchPapers()` | GET | `/api/papers` | 获取论文列表 |
| `addPaper(input)` | POST | `/api/papers` | 添加论文 |
| `getPaper(id)` | GET | `/api/papers/{id}` | 获取论文详情 |
| `deletePaper(id)` | DELETE | `/api/papers/{id}` | 删除论文 |
| `getPdfUrl(id)` | - | `/api/papers/{id}/pdf` | 返回 PDF URL 字符串 |
| `getChatHistory(id, sid)` | GET | `/api/chat/{id}/{sid}/history` | 获取单论文会话历史 |
| `updateChatDraft(id, sid, draft)` | PUT | `/api/chat/{id}/{sid}/history/draft` | 更新单论文会话未发送草稿 |
| `clearChatHistory(id, sid)` | DELETE | `/api/chat/{id}/{sid}/history` | 清空单论文会话历史 |
| `sendMessage(id, sid, msg, quotes?)` | POST | `/api/chat/{id}/{sid}` | 发送消息（SSE） |
| `listCrossPaperSessions()` | GET | `/api/chat/cross-paper/sessions` | 串讲会话列表 |
| `createCrossPaperSession(ids)` | POST | `/api/chat/cross-paper/sessions` | 创建串讲会话 |
| `addPapersToCrossPaperSession(sid, ids)` | PUT | `/api/chat/cross-paper/sessions/{sid}/papers` | 添加论文到串讲 |
| `sendCrossPaperMessage(sid, msg)` | POST | `/api/chat/cross-paper/{sid}` | 串讲对话（SSE） |
| `updateCrossPaperChatDraft(sid, draft)` | PUT | `/api/chat/cross-paper/{sid}/history/draft` | 更新串讲会话未发送草稿 |
| `getConfig()` | GET | `/api/config` | 获取配置 |
| `updateLLMConfig(config)` | PUT | `/api/config/llm` | 更新 LLM 配置 |

`sendMessage()` 和 `sendCrossPaperMessage()` 返回 `AsyncGenerator`，逐行解析 SSE 的 `data:` 行。

## 前端调试与验证约定

- 前端交互问题不要只靠静态读代码推断；只要本地环境允许，应先启动前端并做最小可行自测
- 推荐流程：先阅读代码形成假设，再运行前端 / 连接浏览器或 Electron 复现，修改后先自测核心交互路径，最后再请用户验收
- `npm run build`、lint、单元测试只能证明没有明显编译或类型错误，不能替代滚动、切换、拖拽、焦点、快捷键、动画、异步时序等真实交互测试
- 如果当前环境无法完成自测，需要明确说明已验证和未验证的范围，不能把未验证的交互问题表述成已确认修复
