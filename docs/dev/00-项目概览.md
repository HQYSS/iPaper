# iPaper 开发者文档 - 项目概览

## 项目定位

iPaper 是一个 AI 驱动的 arXiv 论文阅读助手，桌面端应用。用户输入 arXiv 论文地址，系统下载 PDF，左侧显示 PDF 阅读器，右侧提供 LLM 对话面板，可针对论文内容提问。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **桌面壳** | Electron | 加载前端页面，管理后端进程生命周期 |
| **前端** | React 18 + TypeScript + Vite | SPA，运行在 Electron 的 BrowserWindow 中 |
| **样式** | Tailwind CSS + shadcn/ui 风格 | CSS 变量驱动的设计系统 |
| **状态管理** | Zustand | 轻量级 store（paperStore, chatStore, toastStore）|
| **PDF 渲染** | @react-pdf-viewer 3.x | 基于 pdfjs-dist，支持插件机制 |
| **后端** | Python FastAPI | REST API + SSE 流式响应 |
| **LLM** | Gemini 3.1 Pro via OpenRouter | 通过 openai Python SDK 调用 OpenRouter API |
| **数据存储** | 文件系统 + JSON | 无数据库，纯文件 |

## 整体架构

```
┌─────────────────────────────────────────────┐
│                  Electron                    │
│  ┌─────────────────────────────────────────┐ │
│  │         BrowserWindow (前端)            │ │
│  │  ┌──────────┬───────────┬────────────┐  │ │
│  │  │ 论文库   │ PDF 阅读器│ AI 对话    │  │ │
│  │  │ (侧边栏) │ (中间)    │ (右侧面板) │  │ │
│  │  └──────────┴───────────┴────────────┘  │ │
│  └──────────────┬──────────────────────────┘ │
│                 │ HTTP (localhost)            │
│  ┌──────────────▼──────────────────────────┐ │
│  │         FastAPI 后端 (:3000)            │ │
│  │  /api/papers  /api/chat  /api/config    │ │
│  └──────────────┬──────────────────────────┘ │
└─────────────────┼───────────────────────────┘
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
 ~/.ipaper/    arXiv API    Gemini API
 (本地存储)   (论文下载)   (LLM 对话)
```

## 数据流

1. **添加论文**: 前端输入 arXiv ID → POST `/api/papers` → 后端调用 arXiv API 下载 PDF → 保存到 `~/.ipaper/papers/{id}/` → 返回元信息
2. **阅读 PDF**: 前端请求 GET `/api/papers/{id}/pdf` → 后端返回 PDF 文件 → @react-pdf-viewer 渲染
3. **AI 对话**: 前端发送消息 POST `/api/chat/{id}` → 后端将 PDF (base64) + 消息发给 Gemini → SSE 流式返回 → 前端逐字显示
4. **引用交互**: 用户在 PDF 或对话中选中文字 → 点击"引用到对话" → 引用文本出现在输入框 → 发送时一并附上

## 本地数据结构

所有数据存储在 `~/.ipaper/` 下：

```
~/.ipaper/
├── config.json                  # 全局配置（LLM API key 等）
└── papers/
    ├── index.json               # 论文索引（所有已下载论文的列表）
    └── 2301.12345/              # 单篇论文目录（以 arXiv ID 命名）
        ├── meta.json            # 论文元信息
        ├── paper.pdf            # 论文 PDF
        └── chats/               # 多会话目录
            ├── sessions.json    # 会话索引（列表 + last_active_session_id）
            └── s_170000000.json # 单个会话的对话历史
```

### config.json 结构
```json
{
  "llm": {
    "api_base": "https://openrouter.ai/api/v1",
    "api_key": "sk-or-v1-xxx",
    "model": "google/gemini-3.1-pro-preview",
    "temperature": 0.7,
    "max_tokens": 8192
  }
}
```

### meta.json 结构
```json
{
  "arxiv_id": "2301.12345",
  "title": "Paper Title",
  "title_zh": null,
  "summary": "Abstract text...",
  "authors": ["Author A", "Author B"],
  "download_time": "2024-01-15T10:30:00",
  "has_latex": false,
  "translation_status": null,
  "translation_progress": null,
  "pdf_path": "/Users/xxx/.ipaper/papers/2301.12345/paper.pdf"
}
```

## 开发环境启动

### 前置要求
- Python 3.10+ (miniconda/anaconda)
- Node.js 18+（推荐 homebrew 安装）
- npm

### 启动步骤

```bash
# 1. 启动后端（终端 1）
cd backend
pip install -r requirements.txt
python -m uvicorn main:app --host 127.0.0.1 --port 3000

# 2. 启动前端（终端 2）
cd frontend
npm install
npm run dev

# 3. 启动 Electron（终端 3）
cd electron
npm install
./node_modules/electron/dist/Electron.app/Contents/MacOS/Electron . --dev --skip-backend
```

前端开发服务器运行在 `localhost:5173`，通过 Vite proxy 将 `/api` 请求代理到后端 `localhost:3000`。

### 一键启动

双击 `iPaper.app` 即可启动全部服务。启动脚本位于 `iPaper.app/Contents/MacOS/iPaper`。

### Cursor 模式启动

在 Cursor IDE 中使用 iPaper，AI 讲解由 Cursor 原生对话框承担：

```bash
# 一键启动后端 + 前端
./start-cursor-mode.sh

# 在 Cursor Simple Browser 中打开
# http://localhost:5173/?cursor=1
```

Cursor 模式下的对话历史保存：
```bash
# 将 Cursor 自动保存的对话记录按论文整理
python scripts/sync_transcripts.py        # 增量同步
python scripts/sync_transcripts.py --all  # 全量重建
python scripts/sync_transcripts.py --list # 查看可识别的论文对话
```
