## 配置管理 (`config.py`)

使用 `pydantic-settings` 管理配置，支持环境变量（前缀 `IPAPER_`）和配置文件。

```python
class Settings:
    data_dir: Path = ~/.ipaper         # 数据根目录
    host: str = "127.0.0.1"
    port: int = 3000
    llm: LLMConfig                     # LLM 子配置

class LLMConfig:
    api_base: str = "https://openrouter.ai/api/v1"
    api_key: str = ""                  # 需要用户配置（OpenRouter API Key）
    model: str = "google/gemini-3.1-pro-preview"
    temperature: float = 0.7
    max_tokens: int = 8192

class ProfileAnalysisConfig:
    model: str = "anthropic/claude-opus-4.6"  # 共用 llm 的 api_base 和 api_key
    temperature: float = 0.2
    max_tokens: int = 16384
```

- 配置文件：`~/.ipaper/config.json`
- 全局单例：`settings = Settings()`
- 启动时自动创建 `~/.ipaper/` 和 `~/.ipaper/papers/` 目录
- 调用 `settings.save_config()` 持久化到文件

## 数据模型 (`models/schemas.py`)

| 模型 | 用途 |
|------|------|
| `PaperCreate` | 添加论文请求体 (`arxiv_input: str`) |
| `PaperMeta` | 论文完整元信息（存储在 meta.json） |
| `PaperListItem` | 论文列表项（摘要截断到 200 字） |
| `ChatMessage` | 对话消息 (`role: user|assistant`, `content: str`, `quotes?: Quote[]`, `reasoning?: str`) |
| `Quote` | 引用片段 (`text: str`, `source: 'pdf'\|'chat'`) |
| `ChatRequest` | 对话请求体 (`message`, `quotes?: List[Quote]`) |
| `ChatDraft` | 未发送草稿 (`input: str`, `quotes?: List[Quote]`) |
| `ForkData` | 消息分支数据 (`alternatives: List[List[ChatMessage]]`, `active: int`) |
| `ChatHistory` | 对话历史 (`paper_id`, `session_id`, `messages[]`, `forks?`, `draft?`) |
| `ChatHistoryUpdate` | 前端直接更新对话历史请求 (`messages`, `forks?`) |
| `ChatDraftUpdate` | 更新未发送草稿请求 (`draft`) |
| `SessionMeta` | 会话元信息 (`id`, `title`, `created_at`, `updated_at`) |
| `SessionList` | 会话列表 (`sessions[]`, `last_active_session_id`) |
| `SessionCreate` | 新建会话请求 (`title?`) |
| `CrossPaperSessionCreate` | 串讲会话创建请求 (`paper_ids[]`, `title?`) |
| `CrossPaperSessionMeta` | 串讲会话元信息（继承 SessionMeta + `paper_ids[]`）|
| `CrossPaperSessionList` | 串讲会话列表 |
| `CrossPaperAddPapersRequest` | 向串讲添加论文请求 (`paper_ids[]`) |
| `CrossPaperChatRequest` | 串讲对话请求 (`message`, `quotes?`) |
| `CrossPaperChatHistory` | 串讲对话历史 (`session_id`, `paper_ids[]`, `messages[]`, `forks?`) |
| `LLMConfigUpdate` | LLM 配置更新请求 |

## API 接口

### 论文管理 (`/api/papers`)

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/papers` | 论文列表 | - | `PaperListItem[]` |
| POST | `/api/papers` | 添加论文 | `{ arxiv_input: string }` | `PaperMeta` |
| GET | `/api/papers/{id}` | 论文详情 | - | `PaperMeta` |
| DELETE | `/api/papers/{id}` | 删除论文 | - | `{ message }` |
| GET | `/api/papers/{id}/pdf` | 获取 PDF | - | `FileResponse (application/pdf)` |
| GET | `/api/papers/{id}/export` | 导出 PDF | - | `FileResponse (attachment)` |

### 会话管理 (`/api/chat/{paper_id}/sessions`)

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/chat/{id}/sessions` | 会话列表 | - | `SessionList` |
| POST | `/api/chat/{id}/sessions` | 新建会话 | `{ title?: string }` | `SessionMeta` |
| DELETE | `/api/chat/{id}/sessions/{sid}` | 删除会话 | - | `{ message }` |

### 对话 (`/api/chat/{paper_id}/{session_id}`)

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/chat/{id}/{sid}` | 发送消息 | `{ message, quotes?: [{text, source}] }` | SSE 流 |
| GET | `/api/chat/{id}/{sid}/history` | 对话历史 | - | `ChatHistory`（含 `forks`、`draft`） |
| PUT | `/api/chat/{id}/{sid}/history` | 更新历史 | `{ messages, forks? }` | `{ message }` |
| PUT | `/api/chat/{id}/{sid}/history/draft` | 更新未发送草稿 | `{ draft: { input, quotes? } }` | `{ message }` |
| DELETE | `/api/chat/{id}/{sid}/history` | 清空历史 | - | `{ message }` |

**SSE 流格式：**

每行格式为 `data: {JSON}\n\n`，JSON 结构：

```json
// 正常响应片段
{ "type": "chunk", "content": "一段文字" }

// 响应完成
{ "type": "done", "full_response": "完整响应文本" }

// 错误（按异常类型返回中文消息）
{ "type": "error", "message": "API Key 无效或已过期，请在设置中检查" }
// 其他错误消息：API 额度不足或请求过于频繁 / 无法连接到 LLM 服务 / LLM 服务返回错误 (status_code)
```

### 串讲 (`/api/chat/cross-paper`)

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/chat/cross-paper/sessions` | 串讲会话列表 | - | `CrossPaperSessionList` |
| POST | `/api/chat/cross-paper/sessions` | 创建串讲会话 | `{ paper_ids: string[], title? }` | `CrossPaperSessionMeta` |
| DELETE | `/api/chat/cross-paper/sessions/{sid}` | 删除串讲会话 | - | `{ message }` |
| PUT | `/api/chat/cross-paper/sessions/{sid}/papers` | 添加论文到串讲 | `{ paper_ids: string[] }` | `CrossPaperSessionMeta` |
| POST | `/api/chat/cross-paper/{sid}` | 串讲对话 | `{ message, quotes? }` | SSE 流 |
| GET | `/api/chat/cross-paper/{sid}/history` | 串讲对话历史 | - | `CrossPaperChatHistory`（含 `draft`） |
| PUT | `/api/chat/cross-paper/{sid}/history` | 更新串讲历史 | `{ messages, forks? }` | `{ message }` |
| PUT | `/api/chat/cross-paper/{sid}/history/draft` | 更新串讲未发送草稿 | `{ draft: { input, quotes? } }` | `{ message }` |
| DELETE | `/api/chat/cross-paper/{sid}/history` | 清空串讲历史 | - | `{ message }` |

**路由顺序**：cross-paper 路由必须在 `{paper_id}` 路由之前注册，否则 FastAPI 会将 `cross-paper` 作为 `paper_id` 参数匹配。

### 配置 (`/api/config`)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/config` | 获取配置状态（含 `hjfy_cookie_configured`） |
| PUT | `/api/config/llm` | 更新 LLM 配置 |
| PUT | `/api/config/hjfy` | 验证并保存 hjfy.top Cookie |

配置路由统一由 `routers/config.py` 提供。

### 翻译 (`/api/papers/{id}/translate`)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/papers/{paper_id}/translate` | 触发翻译（幂等），返回 `{ status, info, error }` |
| GET | `/api/papers/{paper_id}/translate/status` | 查询翻译进度，`status` 为 `none/pending/polling/finished/failed/error/needs_login` |

翻译路由由 `routers/translation.py` 提供，注册在 `/api/papers` 前缀下。

## 服务层详解

### ArxivService (`services/arxiv_service.py`)

负责论文的下载和本地管理。

**arXiv ID 解析 (`parse_arxiv_input`)：**

支持以下输入格式：
- `2301.12345`
- `2301.12345v1`
- `https://arxiv.org/abs/2301.12345`
- `https://arxiv.org/pdf/2301.12345.pdf`

**下载流程 (`download_paper`)：**
1. 解析 arXiv ID
2. 检查 `~/.ipaper/papers/{id}/meta.json` 是否存在（防重复下载）
3. 使用 `arxiv` 库搜索论文元信息
4. 调用 `paper.download_pdf()` 下载 PDF
5. 生成 `meta.json` 保存元信息
6. 更新 `index.json` 索引

**索引管理：**
- `index.json` 位于 `~/.ipaper/papers/index.json`
- 格式：`{ papers: [{ arxiv_id, title, download_time }] }`
- 添加/删除论文时自动维护

**注意事项：**
- arXiv API 有速率限制（HTTP 429），建议使用 `delay_seconds=3.0`, `num_retries=3`
- 当前代码中的 `arxiv.Client()` 未配置这些参数，频繁操作可能触发限流

### LLMService (`services/llm_service.py`)

负责与 LLM API 通信。

**API 调用方式：**
- 使用 `openai` Python SDK（AsyncOpenAI）
- API 提供商：OpenRouter（`https://openrouter.ai/api/v1`）
- 讲解模型：`google/gemini-3.1-pro-preview`
- Reasoning：通过 `extra_body={"reasoning": {"effort": "medium"}}` 启用 Gemini 思考模式

**PDF 上传方式：**

将 PDF 文件编码为 base64，作为 `file` 类型的消息内容发送（OpenRouter 格式）：

```python
{
    "type": "file",
    "file": {
        "filename": "paper.pdf",
        "fileData": "data:application/pdf;base64,{pdf_base64}"
    }
}
```

**消息构建逻辑 (`_build_messages`)：**
1. 添加 system prompt（学术论文阅读助手角色）
2. 在**第一条用户消息**中附加 PDF（base64）
3. 如果有 `quotes`，在最后一条用户消息前拼接引用文本（带来源标注：`[来自论文]` / `[来自对话]`）
4. assistant 消息附加 `reasoning` 字段（如有），传回 API 保持推理连续性
5. 后续消息正常传递

**System Prompt 要点：**
- 使用中文回答，保留英文术语
- 基于论文内容回答，不编造
- 可引用具体章节和图表

**串讲模式 (`chat_stream_cross_paper`)：**
- 使用 `get_cross_paper_system_prompt()` — 在基础用户画像 prompt 后追加串讲专用指令（对比分析框架、`[[id]]` 引用标注格式）
- `_build_cross_paper_first_user_content()` — 在第一条 user 消息中注入所有论文 PDF（每篇前加 `=== 论文 [[id]]: 标题 ===` 标注），复用 `_build_pdf_content_blocks()` 处理大 PDF
- `_build_messages_cross_paper()` — 构建完整的 API 消息列表

**流式响应 (`chat_stream`)：**

返回 `AsyncGenerator[str, None]`，逐 chunk yield 文本。支持 `reasoning_collector` 参数，在流式过程中静默累积 reasoning tokens，流结束后由调用方（`chat.py`）存入对话历史。

### StorageService (`services/storage_service.py`)

管理多会话对话历史的持久化。

**存储位置：** `papers/{arxiv_id}/chats/`

```
chats/
├── sessions.json         # 会话索引
└── {session_id}.json     # 单个会话历史
```

**sessions.json 结构：**
```json
{
  "sessions": [{ "id": "s_170000000", "title": "自动讲解", "created_at": "...", "updated_at": "..." }],
  "last_active_session_id": "s_170000000"
}
```

**会话文件结构：**
```json
{
  "paper_id": "2301.12345",
  "session_id": "s_170000000",
  "messages": [
    { "role": "user", "content": "这篇论文讲了什么?", "quotes": [{ "text": "...", "source": "pdf" }] },
    { "role": "assistant", "content": "这篇论文提出了...", "reasoning": "（可选）" }
  ],
  "forks": { "1": { "alternatives": [[...], [...]], "active": 1 } },
  "draft": {
    "input": "还没发送的问题",
    "quotes": [{ "text": "...", "source": "chat" }]
  }
}
```

**Session 管理方法：**
- `list_sessions(paper_id)` → `SessionList`
- `create_session(paper_id, title?)` → `SessionMeta`
- `delete_session(paper_id, session_id)` → `bool`
- `set_last_active_session(paper_id, session_id)`

**对话历史方法：**
- `get_chat_history(paper_id, session_id)` → `(List[ChatMessage], forks_dict, draft_dict)`
- `save_chat_history(paper_id, session_id, messages, forks?, draft?)` → 覆盖写入
- `clear_chat_history(paper_id, session_id)` → 删除文件

**引用持久化：** 用户消息上的 `quotes` 会跟随 `ChatMessage` 一起写入历史文件；单论文和串讲接口都采用同一结构，前端刷新后可直接恢复引用块显示

**未发送草稿持久化：** 当前会话的输入框文本和待发送引用保存在同一个会话文件的 `draft` 字段中。前端输入过程中会调用 `/history/draft` 接口同步；切换会话和刷新后通过 `GET /history` 恢复；真正发送消息时会把 `draft` 清空，避免残留旧草稿

**向后兼容：** 首次访问论文时，旧 `chat_history.json` 自动懒迁移到 `chats/` 目录

**串讲存储：** 独立于单论文，位于 `~/.ipaper/cross-paper/`：
- `sessions.json`：串讲会话索引（每个 session 额外存 `paper_ids`）
- `chats/cp_170000000.json`：单个串讲的对话历史
- 方法命名统一加 `cross_paper_` 前缀（如 `list_cross_paper_sessions`、`save_cross_paper_chat_history`）
- `add_papers_to_cross_paper_session(session_id, paper_ids)` 支持向已有串讲添加新论文
