## 配置管理 (`config.py`)

使用 `pydantic-settings` 管理配置，支持环境变量（前缀 `IPAPER_`）和配置文件。

```python
class Settings:
    data_dir: Path = ~/.ipaper         # 数据根目录
    host: str = "127.0.0.1"
    port: int = 3000
    llm: LLMConfig                     # LLM 子配置

class LLMConfig:
    api_base: str = "https://openrouter.ai/api/v1"
    api_key: str = ""                  # 需要用户配置（OpenRouter API Key）
    model: str = "google/gemini-3.1-pro-preview"
    temperature: float = 0.7
    max_tokens: int = 8192

class ProfileAnalysisConfig:
    model: str = "anthropic/claude-opus-4.6"  # 共用 llm 的 api_base 和 api_key
    temperature: float = 0.2
    max_tokens: int = 4096
```

- 配置文件：`~/.ipaper/config.json`
- 全局单例：`settings = Settings()`
- 启动时自动创建 `~/.ipaper/` 和 `~/.ipaper/papers/` 目录
- 调用 `settings.save_config()` 持久化到文件

## 数据模型 (`models/schemas.py`)

| 模型 | 用途 |
|------|------|
| `PaperCreate` | 添加论文请求体 (`arxiv_input: str`) |
| `PaperMeta` | 论文完整元信息（存储在 meta.json） |
| `PaperListItem` | 论文列表项（摘要截断到 200 字） |
| `ChatMessage` | 对话消息 (`role: user|assistant`, `content: str`, `reasoning?: str`) |
| `Quote` | 引用片段 (`text: str`, `source: 'pdf'\|'chat'`) |
| `ChatRequest` | 对话请求体 (`message`, `quotes?: List[Quote]`) |
| `ChatHistory` | 对话历史 (`paper_id`, `messages[]`) |
| `LLMConfigUpdate` | LLM 配置更新请求 |

## API 接口

### 论文管理 (`/api/papers`)

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | `/api/papers` | 论文列表 | - | `PaperListItem[]` |
| POST | `/api/papers` | 添加论文 | `{ arxiv_input: string }` | `PaperMeta` |
| GET | `/api/papers/{id}` | 论文详情 | - | `PaperMeta` |
| DELETE | `/api/papers/{id}` | 删除论文 | - | `{ message }` |
| GET | `/api/papers/{id}/pdf` | 获取 PDF | - | `FileResponse (application/pdf)` |
| GET | `/api/papers/{id}/export` | 导出 PDF | - | `FileResponse (attachment)` |

### 对话 (`/api/chat`)

| 方法 | 路径 | 说明 | 请求体 | 响应 |
|------|------|------|--------|------|
| POST | `/api/chat/{id}` | 发送消息 | `{ message, quotes?: [{text, source}] }` | SSE 流 |
| GET | `/api/chat/{id}/history` | 对话历史 | - | `ChatHistory` |
| DELETE | `/api/chat/{id}/history` | 清空历史 | - | `{ message }` |

**SSE 流格式：**

每行格式为 `data: {JSON}\n\n`，JSON 结构：

```json
// 正常响应片段
{ "type": "chunk", "content": "一段文字" }

// 响应完成
{ "type": "done", "full_response": "完整响应文本" }

// 错误
{ "type": "error", "message": "错误信息" }
```

### 配置 (`/api/config`)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/config` | 获取配置状态 |
| PUT | `/api/config/llm` | 更新 LLM 配置 |

**注意：** `main.py` 中有一个直接定义的 `GET /api/config` 端点（返回简化信息），与 `routers/config.py` 中的路由可能存在冲突，需要注意。

## 服务层详解

### ArxivService (`services/arxiv_service.py`)

负责论文的下载和本地管理。

**arXiv ID 解析 (`parse_arxiv_input`)：**

支持以下输入格式：
- `2301.12345`
- `2301.12345v1`
- `https://arxiv.org/abs/2301.12345`
- `https://arxiv.org/pdf/2301.12345.pdf`

**下载流程 (`download_paper`)：**
1. 解析 arXiv ID
2. 检查 `~/.ipaper/papers/{id}/meta.json` 是否存在（防重复下载）
3. 使用 `arxiv` 库搜索论文元信息
4. 调用 `paper.download_pdf()` 下载 PDF
5. 生成 `meta.json` 保存元信息
6. 更新 `index.json` 索引

**索引管理：**
- `index.json` 位于 `~/.ipaper/papers/index.json`
- 格式：`{ papers: [{ arxiv_id, title, download_time }] }`
- 添加/删除论文时自动维护

**注意事项：**
- arXiv API 有速率限制（HTTP 429），建议使用 `delay_seconds=3.0`, `num_retries=3`
- 当前代码中的 `arxiv.Client()` 未配置这些参数，频繁操作可能触发限流

### LLMService (`services/llm_service.py`)

负责与 LLM API 通信。

**API 调用方式：**
- 使用 `openai` Python SDK（AsyncOpenAI）
- API 提供商：OpenRouter（`https://openrouter.ai/api/v1`）
- 讲解模型：`google/gemini-3.1-pro-preview`
- Reasoning：通过 `extra_body={"reasoning": {"effort": "medium"}}` 启用 Gemini 思考模式

**PDF 上传方式：**

将 PDF 文件编码为 base64，作为 `file` 类型的消息内容发送（OpenRouter 格式）：

```python
{
    "type": "file",
    "file": {
        "filename": "paper.pdf",
        "fileData": "data:application/pdf;base64,{pdf_base64}"
    }
}
```

**消息构建逻辑 (`_build_messages`)：**
1. 添加 system prompt（学术论文阅读助手角色）
2. 在**第一条用户消息**中附加 PDF（base64）
3. 如果有 `quotes`，在最后一条用户消息前拼接引用文本（带来源标注：`[来自论文]` / `[来自对话]`）
4. assistant 消息附加 `reasoning` 字段（如有），传回 API 保持推理连续性
5. 后续消息正常传递

**System Prompt 要点：**
- 使用中文回答，保留英文术语
- 基于论文内容回答，不编造
- 可引用具体章节和图表

**流式响应 (`chat_stream`)：**

返回 `AsyncGenerator[str, None]`，逐 chunk yield 文本。支持 `reasoning_collector` 参数，在流式过程中静默累积 reasoning tokens，流结束后由调用方（`chat.py`）存入对话历史。

---

### StorageService (`services/storage_service.py`)

管理对话历史的持久化。

**存储位置：** `~/.ipaper/papers/{arxiv_id}/chat_history.json`

```json
{
  "paper_id": "2301.12345",
  "messages": [
    { "role": "user", "content": "这篇论文讲了什么?" },
    { "role": "assistant", "content": "这篇论文提出了...", "reasoning": "（模型思考过程，可选）" }
  ]
}
```

**方法：**
- `get_chat_history(paper_id)` → `List[ChatMessage]`
- `save_chat_history(paper_id, messages)` → 覆盖写入
- `append_message(paper_id, message)` → 追加一条
- `clear_chat_history(paper_id)` → 删除文件
